# Cookie Checker Web App

A full-stack web application for extracting and analyzing cookies from browsers or files. Built with React (Vite) frontend and FastAPI backend.

## Features

- **Authentication System**:
  - Secure key-based authentication
  - Master key: `PritongTinola*3030` for administrative access
  - Master can generate login keys for other users
  - Persistent session with localStorage

- **Cookie Parser**: Support for multiple cookie formats:
  - Netscape format (browser dev tools export)
  - JSON format (array of cookie objects)
  - Auto-detection of format
  
- **Cookie Display**: 
  - Comprehensive table view with all cookie properties
  - Easy copy-to-clipboard functionality
  - Download as JSON
  - Raw JSON preview

- **Netflix Token Generation**:
  - Generate Netflix auto-login tokens (NFTOKEN) from cookies
  - Playwright integration to get full cookie headers automatically
  - Support for both complete and partial cookie inputs

- **Account Information**:
  - Extract Netflix account info (when cookies are available)
  - Display plan details, profiles, and membership information

- **Responsive Design**: Works on desktop and mobile devices

## Authentication & Access Control

### How It Works

The Cookie Checker uses a key-based authentication system:

1. **Master Key**: `PritongTinola*3030` provides full access and can generate login keys
2. **Login Keys**: Generated by the master key holder for other users to use
3. **Session Management**: Keys are stored in browser localStorage for persistent sessions
4. **API Key Header**: All protected endpoints require the `X-API-Key` header

### Master Key Access

When accessing the application with the master key:
1. Go to the login page at `http://localhost:5173`
2. Enter the master key: `PritongTinola*3030`
3. Click "Login" to access the application
4. After login, you can choose to generate login keys for users

### Generating Login Keys

As a master key holder:
1. Login with the master key
2. Click "OR" to reveal the management options
3. Click "Manage Keys" to see all existing keys and delete them if needed
4. In the key creation form, enter:
   - User's name (e.g., "John Doe")
   - Custom key/password (e.g., "INDAMO")
5. Click "Create Key" to save it
6. Existing keys are listed with the ability to delete any of them

### Managing Existing Keys

As a master key holder:
1. After login, click "Manage Keys"
2. View all created keys with their user names
3. Delete any key by clicking the "Delete" button
4. Confirmation dialog will appear before deletion

### User Access

Normal users with a login key can:
1. Go to the login page
2. Enter their assigned login key
3. Click "Login" for full access
4. Their name is displayed next to the logout button
5. Session is saved for future visits

### Automatic API Key Injection

Once logged in (as master or user), the frontend automatically adds the `X-API-Key` header to all API requests. No manual configuration needed!

## API Endpoints

### Authentication Endpoints

#### POST `/api/auth/login`

Authenticate with a login key (master key or generated key).

**Request:**
```json
{
  "key": "PritongTinola*3030"
}
```

**Response:**
```json
{
  "success": true,
  "message": "Login successful",
  "is_master": true,
  "user_name": "Admin"
}
```

**Error Response:**
```json
{
  "detail": "Invalid key"
}
```

#### POST `/api/auth/generate-key`

**Request:**

**Response:**
```json
{

  "key": "INDAMO",
  "user_name": "John Doe",

```

**Error Responses:**
```json

```

```json
{
```


### Protected Endpoints (Require API Key)

All protected endpoints require the `X-API-Key` header to be set with any valid login key (master key or user-generated key).

**Example Header:**
```
X-API-Key: PritongTinola*3030
```

Or with a generated key:
```
X-API-Key: generated-random-key-here
```

#### POST `/api/check-cookies`

Parse and extract cookies from provided text.

**Request:**
```json
{
  "cookies_text": "...",
  "format_type": "auto"  // "netscape", "json", or "auto"
}
```

**Response:**
```json
{
  "success": true,
  "cookies": [
    {
      "name": "cookie_name",
      "value": "cookie_value",
      "domain": ".example.com",
      "path": "/",
      "secure": true,
      "httponly": true,
      "samesite": "Strict",
      "expires": "2024-01-01"
    }
  ],
  "count": 1,
  "parsed_at": "2024-03-10T12:00:00",
  "errors": null
}
```

### POST `/api/upload-cookies`

Upload a file containing cookies.

**Parameters:**
- `file`: File upload (text or JSON)

**Response:** Same as `/api/check-cookies`

### POST `/api/generate-netflix-token`

Parse cookies and generate Netflix auto-login token (NFTOKEN).

**Request:**
```json
{
  "cookies_text": "...",
  "format_type": "auto",  // "netscape", "json", or "auto"
  "use_playwright": false  // Use Playwright to get full cookie header
}
```

**Response:**
```json
{
  "success": true,
  "nftoken": "eyJhbGciOiJSUzI1NiIsInR5cCI6IkpXVCJ9...",
  "error": null,
  "cookies": [...],
  "cookie_count": 5
}
```

**Notes:**
- If `use_playwright` is `true`, the endpoint will use Playwright to visit Netflix.com and collect the full cookie header
- Useful when you only have partial cookies (like just NetflixId)
- The generated token can be used for Netflix mobile/TV app authentication
- Requires NetflixId and SecureNetflixId cookies to be present

### GET `/api/stats`

Get API statistics and available endpoints.

**Response:**
```json
{
  "status": "running",
  "timestamp": "2024-03-10T12:00:00",
  "endpoints": [
    "POST /api/check-cookies",
    "POST /api/upload-cookies",
    "POST /api/generate-netflix-token",
    "GET /api/stats"
  ]
}
```

## Cookie Formats

### Netscape Format
Exported from browser DevTools or as tab-separated values:
```
.domain.com	TRUE	/	TRUE	expires	cookieName	cookieValue
```

### JSON Format
Standard JSON array of cookie objects:
```json
[
  {
    "name": "cookieName",
    "value": "cookieValue",
    "domain": ".domain.com",
    "path": "/",
    "secure": true,
    "httpOnly": true,
    "sameSite": "Strict",
    "expires": "2024-01-01"
  }
]
```
